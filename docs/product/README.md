# Product Docs

This folder is the source of truth for Ask AI product decisions.

The best format for this project is a living PRD: one document that records what we are building, why it matters, what is in V1, what is explicitly out of scope, and what we may build later.

## Current Docs

- [Ask AI Product Spec](ask-ai-product-spec.md): V1 product definition, user flows, requirements, non-goals, and future roadmap.

## How To Maintain The Spec

- Keep product decisions concrete and testable.
- Separate V1 scope from future ideas.
- Write non-goals explicitly so the first version stays small enough to ship.
- Add open questions when a decision is not ready yet.
- Move resolved questions into the decision sections instead of leaving stale notes.
- Keep implementation and architecture notes out of this folder until product scope is settled.
